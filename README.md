## 📌 프로젝트 소개

기존 재고관리 시스템은 다음과 같은 문제가 있었습니다.

- 매장–본사 간 재고 데이터 불일치  
- 반복적인 수작업 보고 및 발주  
- 단일 서버 구조로 인한 장애 위험  
- 재고 이상(도난/오류) 탐지 불가  

이를 해결하기 위해 단순 CRUD 시스템이 아닌,  

👉 **MES(Manufacturing Execution System) 구조 기반 재고관리 시스템**으로 재설계하고  
👉 **Kubernetes 기반 확장형 아키텍처 + AI 기능을 결합한 시스템**을 개발했습니다.
